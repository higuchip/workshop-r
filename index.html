Abertura: por que aprender a manipular dados no R?

Profissionais que trabalham com dados florestais e ambientais — seja em consultoria, pesquisa, orgaos ambientais ou empresas do setor florestal — lidam rotineiramente com planilhas extensas: inventarios com milhares de arvores, levantamentos de fauna e flora, monitoramentos de restauracao ecologica, dados de parcelas permanentes. O processamento desses dados (filtrar, calcular, agrupar, resumir) e o passo fundamental antes de qualquer analise ou relatorio.

Na pratica, esse processamento quase sempre e feito manualmente no Excel: copiar colunas, aplicar filtros, arrastar formulas, montar tabelas dinamicas. Funciona, mas e lento, propenso a erros e dificil de repetir. Quando o mesmo trabalho precisa ser refeito para um novo conjunto de dados — outro inventario, outro projeto, outra area — o processo comeca do zero.

O R resolve esse problema de forma direta:

Eficiencia — Operacoes que levam horas no Excel (filtrar milhares de registros, calcular area seccional para cada arvore, gerar tabelas resumo por parcela ou por especie) sao executadas em segundos.

Reutilizacao — Um script criado para processar um inventario pode ser reaproveitado em outros projetos. Basta trocar o arquivo de entrada.

Rastreabilidade — Cada passo esta documentado no codigo. Se um colega precisa conferir como a tabela foi gerada, ou se voce precisa refazer a analise meses depois, o script e o registro completo do que foi feito.

O que este workshop cobre: Este e um workshop de fundamentos de manipulacao de dados. Voce vai aprender a importar dados de inventario florestal, explorar, filtrar, calcular variaveis dendrometricas, agrupar, resumir e exportar tabelas organizadas. Sao as operacoes-base que sustentam qualquer analise florestal.

O que este workshop nao cobre (ainda): Analises fitossociologicas completas (densidade, frequencia, dominancia relativas, IVI), visualizacao grafica com ggplot2, e indices de diversidade sao temas para modulos avancados. Aqui, o foco e dominar a manipulacao — sem ela, nenhuma dessas analises e possivel.

Vamos trabalhar com um inventario florestal real de Floresta Ombrofila Mista como estudo de caso. Ao final do dia, voce sera capaz de pegar uma planilha bruta de campo e transforma-la em tabelas resumo organizadas por parcela e por especie, prontas para alimentar relatorios e analises mais avancadas.

Como usar este material: Abra o RStudio e crie um novo script (File > New File > R Script). Digite os comandos apresentados neste documento no seu script e execute com Ctrl+Enter. Nao copie e cole — digitar ajuda a fixar os comandos! Ao final, voce tera um script completo como referencia para seus projetos.

Modulo 1 — Primeiros passos

1.1 Instalando e carregando o tidyverse

O tidyverse e um conjunto de pacotes criado para facilitar a manipulacao, transformacao e visualizacao de dados no R. O pacote central que usaremos e o dplyr, que fornece uma gramatica intuitiva para manipular tabelas de dados.

Digite no seu script: 
# Instalando o tidyverse (apenas na primeira vez)
install.packages("tidyverse")

# Carregando o tidyverse
library(tidyverse)
Dica: install.packages() so precisa ser executado uma vez no seu computador. Ja o library() deve ser executado toda vez que abrir uma nova sessao do R.

1.2 Importando os dados

Nosso estudo de caso e um inventario florestal em Floresta Ombrofila Mista no planalto serrano de Santa Catarina — um tipo de levantamento comumente exigido em processos de licenciamento que envolvam supressao de vegetacao nativa.

O arquivo inventario.csv contem 1.843 arvores medidas em 50 parcelas, com tres variaveis:

ColunaDescricao
ParcelaNumero da parcela (1 a 50)
EspecieNome cientifico completo da especie
DAPDiametro a altura do peito, em centimetros
Digite no seu script: 
# Verificando o diretorio de trabalho
getwd()

# Se necessario, defina o diretorio:
# setwd("caminho/para/o/diretorio")

# Importando os dados
dados <- read.csv2("inventario.csv")
Atencao: Usamos read.csv2() porque o arquivo usa ponto e virgula (;) como separador — padrao brasileiro. Se o arquivo usasse virgula, a funcao seria read.csv().

Antes de qualquer analise, o primeiro passo e sempre conhecer os dados. Em licenciamento, isso equivale a "conferir a planilha de campo" antes de processar.

Modulo 2 — Manipulacao basica

2.1 O operador pipe: %>%

O pipe (%>%) e o conceito central do tidyverse. Ele encadeia operacoes de forma legivel, passando o resultado de uma funcao como entrada da proxima.

Pense nele como a palavra "entao":

"Pegue os dados, entao filtre, entao selecione, entao ordene."

Digite no seu script: 
# SEM pipe (leitura de dentro para fora)
head(dados, 10)

# COM pipe (leitura da esquerda para a direita)
dados %>% head(10)

Ambos fazem a mesma coisa, mas o pipe torna o codigo mais legivel. A vantagem fica evidente quando encadeamos varias operacoes, como veremos adiante.

Atalho: No RStudio, Ctrl+Shift+M insere %>% automaticamente. Use sempre!

2.2 Selecionando colunas com select()

Em datasets maiores (com dezenas de colunas), voce frequentemente precisa trabalhar apenas com algumas variaveis. O select() escolhe quais colunas manter.

Digite no seu script: 
# Selecionar apenas Especie
dados %>% select(Especie)

# Selecionar Parcela e DAP
dados %>% select(Parcela, DAP)

# Excluir uma coluna (com sinal de menos)
dados %>% select(-Parcela)
Dica: Por enquanto, estamos apenas visualizando os resultados no console. Para salvar, use <- (ex: resultado <- dados %>% select(Especie)).

2.3 Filtrando linhas com filter()

O filter() seleciona linhas que atendem a uma condicao. No dia a dia do licenciamento, e o equivalente a "filtrar a planilha" — mas de forma documentada e reprodutivel.

Filtros numericos

Digite no seu script: 
# Arvores com DAP maior que 30 cm
dados %>% filter(DAP > 30)

# Arvores com DAP entre 10 e 20 cm
dados %>% filter(DAP >= 10 & DAP <= 20)

# Arvores da parcela 1
dados %>% filter(Parcela == 1)
Atencao: Comparacao de igualdade usa == (dois sinais de igual). Operadores logicos: & (E), | (OU), ! (NAO).

Filtros com texto

Situacao pratica: Imagine que voce precisa saber quantos individuos de uma especie ameacada ou de interesse comercial existem no seu inventario. Ou que precisa isolar os dados de uma unica parcela para conferencia. Filtrar por especie ou por parcela e uma das operacoes mais frequentes no processamento de dados florestais.
Digite no seu script: 
# Filtrar Araucaria angustifolia (especie imune ao corte)
dados %>% filter(Especie == "Araucaria angustifolia")

# Filtrar mais de uma especie
dados %>% filter(Especie %in% c("Araucaria angustifolia",
                                 "Podocarpus lambertii"))

Combinando operacoes com pipe

Digite no seu script: 
# Especie e DAP das arvores com DAP > 40 cm
dados %>%
  filter(DAP > 40) %>%
  select(Especie, DAP)

2.4 Ordenando com arrange()

Digite no seu script: 
# Ordenar por DAP crescente
dados %>% arrange(DAP)

# Ordenar por DAP decrescente
dados %>% arrange(desc(DAP))

# Ordenar por Parcela, depois por DAP decrescente
dados %>% arrange(Parcela, desc(DAP))
Exercicio 3: Quais sao as 10 maiores arvores do inventario? Mostre Especie, Parcela e DAP, ordenadas do maior para o menor DAP.
Dica: combine arrange(), select() e head() com o pipe.

Modulo 3 — Transformacao de dados

3.1 Criando novas colunas com mutate()

O mutate() cria novas colunas a partir de calculos com as existentes. No contexto de inventario florestal, frequentemente precisamos calcular variaveis dendrometricas a partir das medidas de campo.

Situacao pratica: Os dados brutos de campo geralmente trazem apenas as medidas diretas (DAP, altura, circunferencia). Mas para qualquer analise florestal — seja um relatorio de inventario, um plano de manejo ou um estudo academico — voce precisa calcular variaveis derivadas: area seccional, volume, classes de diametro. O mutate() faz isso para todas as arvores de uma so vez.

Area seccional

A area seccional (g) e a area da secao transversal do tronco, calculada pela formula:

g = π × DAP² / 40000   (resultado em m²)

Digite no seu script: 
# Calculando a area seccional (m2)
dados %>%
  mutate(g = pi * DAP^2 / 40000)

Volume estimado

Uma estimativa simplificada de volume individual pode ser obtida com um fator de forma:

vol = g × h × ff

Onde h e a altura (usaremos 10 m como estimativa media) e ff e o fator de forma (usaremos 0,7, valor tipico para florestas nativas):

Digite no seu script: 
# Volume estimado (m3)
dados %>%
  mutate(
    g   = pi * DAP^2 / 40000,
    vol = g * 10 * 0.7
  )

Classificacao por classe de diametro

A classificacao em classes de diametro e exigida em praticamente todo inventario florestal para licenciamento. Usamos case_when():

Digite no seu script: 
# Classes de diametro
dados %>%
  mutate(classe_dap = case_when(
    DAP < 10             ~ "5-10",
    DAP >= 10 & DAP < 20 ~ "10-20",
    DAP >= 20 & DAP < 30 ~ "20-30",
    DAP >= 30 & DAP < 40 ~ "30-40",
    DAP >= 40 & DAP < 50 ~ "40-50",
    DAP >= 50            ~ "50+"
  ))

3.2 Salvando as transformacoes

Vamos salvar todas as variaveis calculadas em um novo objeto:

Digite no seu script: 
# Criando dataset completo com variaveis calculadas
dados2 <- dados %>%
  mutate(
    g         = pi * DAP^2 / 40000,
    vol       = g * 10 * 0.7,
    classe_dap = case_when(
      DAP < 10             ~ "5-10",
      DAP >= 10 & DAP < 20 ~ "10-20",
      DAP >= 20 & DAP < 30 ~ "20-30",
      DAP >= 30 & DAP < 40 ~ "30-40",
      DAP >= 40 & DAP < 50 ~ "40-50",
      DAP >= 50            ~ "50+"
    )
  )

# Conferindo
head(dados2)
glimpse(dados2)

Modulo 4 — Agrupamento e resumo

Ate agora, trabalhamos com registros individuais (cada linha = uma arvore). Neste modulo, vamos aprender a resumir dados por grupos — a operacao mais importante para gerar tabelas tecnicas de relatorios.

4.1 Contando com count()

Digite no seu script: 
# Quantas arvores por parcela?
dados %>% count(Parcela)

# Quantas arvores por especie? (ordenado)
dados %>%
  count(Especie) %>%
  arrange(desc(n))

# Quantas arvores por classe de diametro?
dados2 %>% count(classe_dap)
Dica: count() cria automaticamente uma coluna n com a contagem. Voce pode renomear com count(Especie, name = "abundancia").
Exercicio 5:
a) Quantas especies distintas foram registradas no inventario?
Dica: conte por especie e depois use nrow().
b) Qual a especie mais abundante? E a menos abundante?

4.2 Agrupando com group_by() + summarise()

A combinacao group_by() + summarise() e a ferramenta mais poderosa do dplyr. Ela permite calcular qualquer estatistica para cada grupo.

Situacao pratica: Praticamente toda analise florestal exige resumir os dados por algum agrupamento: numero de arvores por parcela, area basal por especie, volume por classe de diametro. No Excel, isso seria feito com tabelas dinamicas. No R, group_by() + summarise() faz o mesmo — de forma mais rapida, flexivel e documentada.
Digite no seu script: 
# Resumo por parcela: numero de arvores e DAP medio
dados %>%
  group_by(Parcela) %>%
  summarise(
    n_arvores = n(),
    dap_medio = mean(DAP)
  )

Leia como: "Pegue os dados, entao agrupe por parcela, entao resuma calculando o numero de arvores e o DAP medio para cada parcela."

Funcoes uteis dentro de summarise():
n() — numero de observacoes
n_distinct() — numero de valores unicos
mean() — media
sd() — desvio padrao
min(), max() — minimo e maximo
median() — mediana
sum() — soma

4.3 Tabela resumo por parcela para relatorio tecnico

Digite no seu script: 
# Tabela completa por parcela
resumo_parcela <- dados2 %>%
  group_by(Parcela) %>%
  summarise(
    n_ind      = n(),
    n_especies = n_distinct(Especie),
    dap_medio  = round(mean(DAP), 2),
    dap_max    = round(max(DAP), 2),
    G          = round(sum(g), 4),
    vol_total  = round(sum(vol), 4)
  )

# Visualizando
resumo_parcela

# Visualizando como planilha
View(resumo_parcela)
Dica: A funcao round(x, 2) arredonda para 2 casas decimais — util para tabelas de relatorio.

4.4 Tabela resumo por especie

A lista de especies com abundancia, frequencia e dominancia e peca central de qualquer estudo floristico para licenciamento:

Digite no seu script: 
# Tabela por especie
resumo_especie <- dados2 %>%
  group_by(Especie) %>%
  summarise(
    abundancia  = n(),
    n_parcelas  = n_distinct(Parcela),
    dap_medio   = round(mean(DAP), 2),
    dap_max     = round(max(DAP), 2),
    G_total     = round(sum(g), 4),
    vol_total   = round(sum(vol), 4)
  ) %>%
  arrange(desc(abundancia))

# As 15 especies mais abundantes
resumo_especie %>% head(15)
Exercicio 6:
a) Qual parcela tem a maior area basal (G)?
b) Qual parcela apresenta a maior riqueza de especies?
c) Quantas especies ocorrem em apenas 1 parcela? Essas seriam as especies raras do levantamento.
d) Qual especie apresenta o maior volume total estimado?

4.5 Combinando filter com group_by

Podemos filtrar antes ou depois de agrupar. A ordem muda o resultado:

Digite no seu script: 
# Filtrar ANTES: resumo apenas das arvores com DAP >= 10
dados2 %>%
  filter(DAP >= 10) %>%
  group_by(Parcela) %>%
  summarise(
    n_arvores = n(),
    dap_medio = round(mean(DAP), 2)
  )

# Filtrar DEPOIS: apenas parcelas com mais de 50 arvores
dados2 %>%
  group_by(Parcela) %>%
  summarise(n_arvores = n()) %>%
  filter(n_arvores > 50)
A ordem importa! Filtrar antes de agrupar remove linhas dos dados brutos. Filtrar depois de agrupar remove grupos do resumo.

4.6 Distribuicao por classe de diametro

Situacao pratica: A distribuicao diametrica e uma das tabelas mais comuns em qualquer relatorio de inventario florestal — seja para fins academicos, planos de manejo ou estudos ambientais. Veja como gera-la em poucas linhas:
Digite no seu script: 
# Distribuicao diametrica geral
dados2 %>%
  count(classe_dap, name = "n_arvores") %>%
  arrange(classe_dap)

# Distribuicao diametrica por especie (exemplo: Araucaria)
dados2 %>%
  filter(Especie == "Araucaria angustifolia") %>%
  count(classe_dap, name = "n_arvores") %>%
  arrange(classe_dap)

4.7 Exportando resultados

Ao final do processamento, voce pode exportar as tabelas em CSV para usar no Word, Excel ou outro software de edicao de relatorios:

Digite no seu script: 
# Exportando tabelas
write.csv2(resumo_parcela, "resumo_por_parcela.csv", row.names = FALSE)
write.csv2(resumo_especie, "resumo_por_especie.csv", row.names = FALSE)
Dica: write.csv2() gera um CSV com ponto e virgula como separador e virgula decimal — abre corretamente no Excel em portugues.
